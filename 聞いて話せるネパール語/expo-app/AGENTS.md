# v1.6.0: バイリンガル UI

## 実装内容
- 日本語 / ネパール語の UI 切替対応
- `expo-localization` でシステム言語自動検出
- 設定画面に手動切替 UI
- 翻訳ファイル: `src/i18n/ja.json` (日本語) / `src/i18n/ne.json` (ネパール語)
- 全 UI 文字列、テーマ名 30 個、文法分野名 30 個、単語カテゴリ 30 個を翻訳

## 翻訳の更新方法
- `src/i18n/ja.json` の構造をベースに `ne.json` を編集
- キーが見つからない場合は ja に自動フォールバック
- 開発時にキー文字列が画面に出れば、翻訳漏れがあるサイン

# 過去計画: RNTP 移行（v2.0 で実施予定）

## 経緯
- v1.5.13: 事前結合 MP3 方式で Android バックグラウンド再生を実現（→ Android クラッシュで撤回）
- v1.5.14-1.5.17: 一時的に試行錯誤したが、最終的に個別 MP3 方式に戻った
- v1.5.17 で確認できた事実:
  - **react-native-track-player v5.0.0-alpha0** は当環境 (Expo SDK 54 + RN 0.81 + newArchEnabled=true) で **iOS / Android 両方ビルド成功**
  - つまり将来の RNTP 移行は技術的に実現可能

## RNTP 移行を提案すべきタイミング
- v2.0 として大型アップデートを出すタイミング
- またはユーザーから「シャッフルやバックグラウンドが完璧に動いてほしい」要望が複数件
- アプリ容量を減らしたい時 (結合方式でなくなったが、念のため)

## 移行手順の見通し
1. `npm install react-native-track-player@5.0.0-alpha0` または最新の @rntp/player
2. `PlaybackService` 登録 (index.ts または App.tsx)
3. ListeningScreen を RNTP API に書き換え（個別 MP3 ファイルベースのまま）
4. PracticeScreen も RNTP に移行（expo-audio を全廃すれば一貫性アップ）
5. AGENTS.md を更新

## 注意点
- v5.0.0-alpha0 は alpha 品質、長期サポートなし（メンテナーが有料 @rntp/player に移行済み）
- 商用継続性が重要なら @rntp/player ($99/月) も選択肢
- 移行は段階的に（聞き流しモードのみ先に移行 → 動作確認 → 練習モードへ拡大 の順）
